# 16 常见问题与排查

## 启动失败

| 现象 | 排查命令 | 处理 |
|---|---|---|
| `lightrag-server` 不存在 | `which lightrag-server` | 运行 `uv sync --extra api` 并激活 `.venv`。 |
| import error | `uv sync --extra api` | 缺依赖，按需要加 `--extra offline`。 |
| `.env` 缺失 | `test -f .env && echo ok || echo missing` | 复制 `env.example` 为 `.env` 后修改。 |
| 配置解析失败 | `lightrag-server` 查看日志 | 检查 `LLM_BINDING`、`EMBEDDING_BINDING`、`LIGHTRAG_PARSER`。 |

不要把 `.env` 内容贴到公开日志或文档中。

## 端口占用

```bash
ss -ltnp | grep 9621
```

处理方式：

- 停止占用进程；
- 修改 `.env` 中 `PORT`；
- Docker 用 `PORT=<新端口> docker compose up -d`。

## WebUI 打不开

| 排查 | 命令 |
|---|---|
| 后端是否启动 | `curl http://localhost:9621/health` |
| WebUI 构建产物是否存在 | `find lightrag/api/webui -maxdepth 2 -type f | head` |
| 根路径是否跳转 | `curl -I http://localhost:9621/` |
| 日志是否报静态文件错误 | `tail -n 200 lightrag.log` |

解决：

```bash
cd lightrag_webui
bun install --frozen-lockfile
bun run build
cd ..
lightrag-server
```

## 文档上传失败

| 原因 | 排查 |
|---|---|
| 文件类型不支持 | 看 `lightrag/api/routers/document_routes.py::DocumentManager.supported_extensions` 和前端 `supportedFileTypes`。 |
| 文件过大 | 检查 `MAX_UPLOAD_SIZE`。 |
| 文件名重复 | 查看 `/documents/paginated` 是否已有同源文件。 |
| pipeline 正在 destructive job | 查看 `/documents/pipeline_status`。 |
| parser hint 错误 | 检查文件名 `.[engine-options].ext` 和 `LIGHTRAG_PARSER`。 |

命令：

```bash
curl http://localhost:9621/documents/pipeline_status
curl -X POST http://localhost:9621/documents/paginated \
  -H "Content-Type: application/json" \
  -d '{"page":1,"page_size":20}'
```

## 文档一直 processing

排查：

```bash
curl http://localhost:9621/documents/pipeline_status
curl http://localhost:9621/health
tail -n 300 lightrag.log
```

常见原因：

| 原因 | 说明 |
|---|---|
| LLM 调用卡住 | 检查 LLM host/key/model、`LLM_TIMEOUT`。 |
| Embedding 服务慢或不可达 | 检查 `EMBEDDING_BINDING_HOST`、模型是否已拉取。 |
| Parser 外部服务不可达 | MinerU/Docling endpoint 未配置或不可达。 |
| 并发队列积压 | `/health` 中查看 `llm_queue_status`、`embedding_queue_status`。 |
| 文档太大 | 调整 chunk 和 parser 参数，或拆分文档。 |

可以尝试取消：

```bash
curl -X POST http://localhost:9621/documents/cancel_pipeline
```

## Embedding 维度不匹配

现象：

- 向量库 upsert/query 报 dimension mismatch；
- 更换模型后检索结果异常。

排查：

```bash
curl http://localhost:9621/health
```

查看 `configuration.embedding_model`、`EMBEDDING_DIM`。

处理：

1. 确认模型真实输出维度。
2. 设置正确 `EMBEDDING_DIM`。
3. 清空旧向量数据并重新索引。

默认本地要重点清理：

```bash
rm rag_storage/vdb_chunks.json rag_storage/vdb_entities.json rag_storage/vdb_relationships.json
```

实际删除前确认目录和备份。

## LLM API 调用失败

| 排查 | 命令/说明 |
|---|---|
| host 是否可达 | `curl <LLM_BINDING_HOST>`，不要带真实 key 输出。 |
| 模型名是否正确 | 查看 provider 控制台或本地模型列表。 |
| API Key 是否注入 | `printenv | grep LLM_BINDING`，注意不要公开输出。 |
| Server 配置快照 | `curl http://localhost:9621/health` |
| 角色模型是否覆盖 | `/health` 中 `role_llm_config` |

Ollama：

```bash
ollama list
curl http://localhost:11434/api/tags
```

## Reranker 调用失败

排查：

```bash
curl http://localhost:9621/health
```

看：

- `enable_rerank`
- `rerank_binding`
- `rerank_model`
- `rerank_binding_host`
- `rerank_queue_status`

如果未配置 reranker，可以在查询中设置：

```json
{"enable_rerank": false}
```

或前端查询设置中关闭 rerank。

## 查询没有结果

| 原因 | 排查 |
|---|---|
| 文档未处理完成 | `/documents/status_counts` |
| 查询模式不适合 | 尝试 `naive`、`hybrid`、`mix`。 |
| `COSINE_THRESHOLD` 太高 | 降低阈值。 |
| `top_k` / `chunk_top_k` 太小 | 提高参数。 |
| Embedding 模型不匹配 | 清空重建索引。 |
| KG 抽取失败 | 查看日志中 `extract_entities` 错误。 |

命令：

```bash
curl http://localhost:9621/documents/status_counts
curl -X POST http://localhost:9621/query \
  -H "Content-Type: application/json" \
  -d '{"query":"测试问题","mode":"naive","enable_rerank":false}'
```

## 图谱为空

常见原因：

| 原因 | 说明 |
|---|---|
| 使用 `!` 跳过 KG | process options 包含 `!` 时不抽取实体关系。 |
| 文档还未 processed | 查看状态。 |
| 抽取模型失败 | 查看 LLM 日志和 `Chunk extracted` 前后日志。 |
| Prompt 不适合语言/领域 | 调整 `SUMMARY_LANGUAGE`、Prompt profile。 |
| 查询 label 不存在 | WebUI 图谱 label 先用 `*`。 |

命令：

```bash
curl "http://localhost:9621/graph/label/list"
curl "http://localhost:9621/graphs?label=*&max_depth=3&max_nodes=100"
```

## Docker 启动失败

```bash
docker compose ps
docker compose logs lightrag
docker compose config
```

常见问题：

| 问题 | 处理 |
|---|---|
| `.env` 没挂载 | 确认 `./.env` 存在。 |
| 端口占用 | 修改 `PORT`。 |
| volume 权限 | 检查 `./data` 权限。 |
| 访问宿主 Ollama 失败 | 使用 `host.docker.internal`。 |
| full compose 依赖未健康 | 查看 postgres/neo4j/milvus/vLLM 日志。 |

## WSL 下 localhost 访问问题

| 现象 | 处理 |
|---|---|
| Windows 浏览器打不开 WSL 服务 | 确认 Server 监听 `0.0.0.0`，再访问 `http://localhost:9621`。 |
| Docker 容器访问 Windows/Ollama | 使用 `host.docker.internal`。 |
| 文件性能慢 | 尽量把项目和 `rag_storage` 放在 WSL ext4 路径，不要放 `/mnt/c`。 |
| 端口被 Windows 进程占用 | Windows PowerShell 查端口或改 `PORT`。 |

## Git / uv.lock / .env 注意事项

| 文件 | 建议 |
|---|---|
| `.env` | 不提交，不输出真实内容。 |
| `uv.lock` | 只有依赖变更时才提交；文档任务不应改它。 |
| `lightrag_webui/bun.lock` | 只有前端依赖变更时才提交。 |
| `rag_storage/`、`data/`、`inputs/`、`logs/` | 运行数据，不应提交。 |
| `__pycache__/`、`.pytest_cache/`、`node_modules/`、`.venv/` | 缓存/依赖，不应提交。 |

检查：

```bash
git status --short
git diff --stat
```

flowchart TD

  D -->|文档| E[/documents/pipeline_status + logs]
  D -->|查询| F[/documents/status_counts + /query naive]
