# 17 源码调用链路索引

## 从 `lightrag-server` 命令开始的启动链路

```mermaid
flowchart TD
  A[pyproject.toml: lightrag-server] --> B[lightrag.api.lightrag_server:main]
  B --> C[initialize_config]
  C --> D[parse_args in api/config.py]
  B --> E[check_env_file]
  B --> F[configure_logging]
  B --> G[create_app]
  G --> H[create_optimized_embedding_function]
  G --> I[create_llm_model_func]
  G --> J[LightRAG.__post_init__]
  G --> K[FastAPI lifespan]
  K --> L[rag.initialize_storages]
  K --> M[rag.check_and_migrate_data]
  G --> N[include routers]
  B --> O[uvicorn.run]
```

文件/函数索引：

| 顺序 | 文件 | 函数 |
|---|---|---|
| 1 | `pyproject.toml` | `[project.scripts]` |
| 2 | `lightrag/api/lightrag_server.py` | `main` |
| 3 | `lightrag/api/config.py` | `parse_args` |
| 4 | `lightrag/api/lightrag_server.py` | `create_app` |
| 5 | `lightrag/lightrag.py` | `LightRAG.__post_init__` |
| 6 | `lightrag/lightrag.py` | `initialize_storages` |

## 从上传文档接口开始的索引链路

```mermaid
flowchart TD
  A[POST /documents/upload] --> B[upload_to_input_dir]
  B --> C[_reserve_enqueue_slot]
  B --> D[pipeline_index_file background]
  D --> E[pipeline_enqueue_file]
  E --> F[resolve_file_parser_directives]
  E --> G[rag.apipeline_enqueue_documents]
  G --> H[full_docs.upsert]
  G --> I[doc_status.upsert PENDING]
  D --> J[rag.apipeline_process_enqueue_documents]
  J --> K[_run_pipeline_batch]
  K --> L[_parse_worker]
  L --> M[parse_native/mineru/docling]
  M --> N[_analyze_worker]
  N --> O[process_single_document]
  O --> P[chunks_vdb.upsert]
  O --> Q[_process_extract_entities]
  Q --> R[operate.extract_entities]
  R --> S[operate.merge_nodes_and_edges]
  S --> T[_insert_done]
```

涉及文件：

| 阶段 | 文件 | 函数/类 |
|---|---|---|
| API | `lightrag/api/routers/document_routes.py` | `upload_to_input_dir`、`pipeline_index_file`、`pipeline_enqueue_file` |
| 路由解析 | `lightrag/parser/routing.py` | `resolve_file_parser_directives`、`parse_process_options` |
| enqueue | `lightrag/pipeline.py` | `apipeline_enqueue_documents` |
| process | `lightrag/pipeline.py` | `apipeline_process_enqueue_documents`、`_run_pipeline_batch`、`process_single_document` |
| 抽取 | `lightrag/lightrag.py` / `lightrag/operate.py` | `_process_extract_entities`、`extract_entities` |
| 合并 | `lightrag/operate.py` | `merge_nodes_and_edges` |
| 持久化 | `lightrag/lightrag.py` | `_insert_done` |

## 从 query 接口开始的查询链路

```mermaid
flowchart TD
  A[POST /query] --> B[query_routes.py::query_text]
  A2[POST /query/stream] --> C[query_text_stream]
  B --> D[QueryRequest.to_query_params]
  C --> D
  D --> E[rag.aquery_llm]
  E --> F{mode}
  F -->|local/global/hybrid/mix| G[operate.kg_query]
  F -->|naive| H[operate.naive_query]
  F -->|bypass| I[role_llm_funcs.query]
  G --> J[get_keywords_from_query]
  J --> K[extract_keywords_only]
  G --> L[_build_query_context]
  L --> M[_perform_kg_search]
  M --> N[entities_vdb / relationships_vdb / chunks_vdb]
  L --> O[process_chunks_unified]
  O --> P[apply_rerank_if_enabled]
  L --> Q[query role LLM]
```

涉及文件：

| 阶段 | 文件 | 函数 |
|---|---|---|
| API | `lightrag/api/routers/query_routes.py` | `query_text`、`query_text_stream`、`query_data` |
| 参数 | `lightrag/base.py` | `QueryParam` |
| Core | `lightrag/lightrag.py` | `aquery_llm`、`aquery_data` |
| 查询算法 | `lightrag/operate.py` | `kg_query`、`naive_query` |
| rerank | `lightrag/utils.py` / `lightrag/rerank.py` | `process_chunks_unified`、`apply_rerank_if_enabled`、`cohere_rerank`、`jina_rerank`、`ali_rerank` |

## 从 LLM 配置到实际调用模型的链路

```mermaid
flowchart TD
  A[env: LLM_BINDING/LLM_MODEL/HOST/API_KEY] --> B[api/config.py::parse_args]
  B --> C[lightrag_server.py::create_llm_model_func]
  C --> D[create_role_llm_func for each role]
  D --> E[RoleLLMConfig]
  E --> F[LightRAG.__post_init__]
  F --> G[_RoleLLMMixin._rebuild_role_llm_funcs]
  G --> H[role_llm_funcs.extract/keyword/query/vlm]
  H --> I[operate.extract_entities / extract_keywords_only / kg_query]
  I --> J[lightrag/llm/provider.py]
```

| 配置 | 解析/使用 |
|---|---|
| `LLM_BINDING` | `create_llm_model_func` 和 role fallback。 |
| `EXTRACT_LLM_BINDING` | `resolve_role_llm_settings("extract")`。 |
| `KEYWORD_LLM_MODEL` | keyword 角色模型。 |
| `QUERY_LLM_TIMEOUT` | query 角色超时。 |
| `VLM_PROCESS_ENABLE` | 多模态处理开关。 |

## 从 Embedding 配置到实际生成向量的链路

```mermaid
flowchart TD
  A[env: EMBEDDING_BINDING/MODEL/DIM/HOST/API_KEY] --> B[api/config.py::parse_args]
  B --> C[lightrag_server.py::create_optimized_embedding_function]
  C --> D[EmbeddingFunc]
  D --> E[LightRAG.__post_init__ wraps with priority/timeout]
  E --> F[chunks_vdb/entities_vdb/relationships_vdb]
  F --> G[VectorStorage.upsert/query]
  G --> H[provider embed function]
```

调用点：

| 场景 | 文件 | 函数 |
|---|---|---|
| chunk 写入 | `lightrag/pipeline.py` | `process_single_document` -> `chunks_vdb.upsert` |
| entity 写入 | `lightrag/operate.py` | `_merge_nodes_then_upsert` |
| relation 写入 | `lightrag/operate.py` | `_merge_edges_then_upsert` |
| query 检索 | `lightrag/kg/*_impl.py` | `query` |

## 从删除文档接口到存储更新的链路

```mermaid
flowchart TD
  A[DELETE /documents/delete_document] --> B[document_routes.py::delete_document]
  B --> C[background_delete_documents]
  C --> D[rag.adelete_by_doc_id]
  D --> E[删除 full_docs/doc_status/text_chunks]
  D --> F[删除 chunks_vdb]
  D --> G[更新 entity/relation/graph]
  G --> H[_insert_done]
```

具体删除细节以 `lightrag/lightrag.py::adelete_by_doc_id` 和相关 helper 为准。

## 从图谱编辑接口到 Core 的链路

| Endpoint | Core |
|---|---|
| `POST /graph/entity/edit` | `rag.aedit_entity` |
| `POST /graph/relation/edit` | `rag.aedit_relation` |
| `POST /graph/entity/create` | `rag.acreate_entity` |
| `POST /graph/relation/create` | `rag.acreate_relation` |
| `POST /graph/entities/merge` | `rag.amerge_entities` |

这些接口会同时维护 Graph、Vector、KV 中的相关数据。修改前建议读 `lightrag/api/routers/graph_routes.py` 和 `lightrag/lightrag.py` 对应方法。
